# 实验计划：前缀修正提升下游可预测性 (H2)


## 核心假设

在多步推理轨迹中，修正早期步骤（尤其至第2步）能够：
1. 恢复后续步骤的正向信息增益
2. 提高从后续步骤隐藏状态预测最终正确性的能力

即使后续步骤仍由模型自主生成。

---

## 整体架构

```
GSM8K 题目
    │
    ├── 开源模型生成原始 CoT（含步骤边界标注）
    │
    ├── 标记最终答案正确/错误
    │
    ├── 前缀干预（替换 s1 / s1-s2 / s1-s3 / s1-s4）
    │       └── 从第 k+1 步起由模型重新生成续写
    │
    └── 三条评估路线并行
            ├── 路线 A：隐藏状态探针
            ├── 路线 B：前缀条件下的可分离性变化
            └── 路线 C：逐步信息增益
```

---

## 阶段一：数据准备

### 1.1 模型选择

- **主实验模型**：开源推理模型（如 DeepSeek-Math、Qwen2.5-Math、Llama-3 等）
- 要求：能提取各层 hidden states，能在步骤边界处截取表征

### 1.2 生成原始 CoT

- 数据集：GSM8K（约 1319 道测试题）
- 每道题采样 N 条推理轨迹（建议 N=8~16，用于增加统计量）
- 标注步骤边界（按换行符 / "Step X" / "\n\n" 等模型输出格式切分）
- 标记每条轨迹的最终答案正确性

### 1.3 前缀修正

对每道题的**错误轨迹**，构造修正版本：

| 干预条件 | 替换内容 | 续写方式 |
|---------|---------|---------|
| Corrected-1 | 替换 $s_1$ | 模型从 $s_2$ 起重新生成 |
| Corrected-2 | 替换 $s_1, s_2$ | 模型从 $s_3$ 起重新生成 |
| Corrected-3 | 替换 $s_1, s_2, s_3$ | 模型从 $s_4$ 起重新生成 |
| Corrected-4 | 替换 $s_1 \dots s_4$ | 模型从 $s_5$ 起重新生成 |
| Control | 长度匹配但语义无关的前缀 | 模型重新生成 |

**construct_corrected_prefix_by_gpt提供正确的前 k 步
2. 同模型多次采样中挑选正确前缀（best-of-N）
3. PRM 打分最高的候选前缀

**Control 前缀**：长度匹配、格式相同，但内容为另一道题的正确步骤（语义无关），用于隔离"语义修正"与"泛化前缀效应"。

---

## 阶段二：路线 A — 隐藏状态探针

### 目标
在每一步的隐藏状态上训练探针，预测该轨迹最终是否正确。

### 步骤

1. **提取 hidden states**
   - 对每条轨迹（原始 + 各修正版本），前向传播，提取每个步骤边界处最后一个 token 的隐藏状态
   - 提取层：最后一层、中间层、每隔 N 层（视模型大小决定粒度）

2. **训练探针**
   - 输入：步骤 t 处的 hidden state 向量
   - 标签：该轨迹最终答案是否正确（二分类）
   - 探针类型：logistic regression（主）、线性 SVM、浅层 MLP（验证）
   - 训练/测试划分：按题目 ID 划分（避免同题泄露）

3. **评估指标**
   - 每步探针的 AUC、准确率
   - 校准度：ECE、Brier Score
   - 绘制：AUC vs 步骤位置曲线

---

## 阶段三：路线 B — 前缀条件下的可分离性变化

### 目标
验证修正早期步骤后，**后续步骤**的探针性能是否显著提升。

### 步骤

1. 对同一道题的同一条错误轨迹，分别在"原始前缀"和"Corrected-2 前缀"条件下提取后续步骤（步骤 3 及之后）的隐藏状态

2. 分别训练/评估探针，计算后续步骤的 AUC

3. **核心检验统计量 — 双重差分**：

$$\Delta = \big(\text{AUC}_{\text{后期}|\text{修正}} - \text{AUC}_{\text{后期}|\text{原始}}\big) - \big(\text{AUC}_{\text{早期}|\text{修正}} - \text{AUC}_{\text{早期}|\text{原始}}\big)$$

如果 $\Delta > 0$ 且显著，说明修正**专门提升了下游可预测性**，而非仅仅平移了所有步骤。

### 统计检验
- DeLong 检验 / Bootstrap：AUC 差异的显著性（按题目配对）
- 置换检验：在配对样本内打乱"修正/原始"标签，重新计算 $\Delta$，得到零分布

### 可视化
- **隐藏状态可分离性热力图**：层（y轴）× 步骤（x轴），颜色 = 探针 AUC
- 画两张：原始前缀 vs 修正前缀，对比展示

---

## 阶段四：路线 C — 逐步信息增益 (IG)

### 目标
量化每一步对正确答案贡献了多少"有用信息"，验证第2步是信息崩塌点，修正后可恢复。

### 步骤

1. **训练监督模型**
   - 输入：题目 + 推理链截至第 t 步的前缀
   - 输出：预测正确最终答案
   - 可直接用同一开源模型 + few-shot prompting，或微调一个小模型

2. **计算逐步 IG**
   - 对每条轨迹、每一步 t：
   
   $$\text{IG}(t) = -\log P(\text{正确答案} \mid \text{前缀}_{1:t}) + \log P(\text{正确答案} \mid \text{前缀}_{1:t-1})$$
   
   即第 t 步带来的对数似然变化量。
   - IG > 0：该步增加了有用信息
   - IG < 0：该步引入了误导信息
   - IG ≈ 0：该步无信息贡献

3. **对比 IG 曲线**
   - 正确轨迹 vs 错误轨迹的 IG 曲线
   - 错误轨迹原始 vs Corrected-2 修正后的 IG 曲线

### 可视化
- 双图对齐：上图逐步正确率，下图逐步 IG
- 标注第2步处 IG 的崩塌与恢复

## 阶段五：混杂控制

| 混杂因素 | 控制方法 |
|---------|---------|
| 步骤长度不均 | 按 token 数归一化；报告每步平均 token 数 |
| 碰撞偏差 | 同时报告无条件曲线和条件曲线（给定最终错误） |
| 事后合理化 | 截断实验：在第 t 步截断，看模型能否直接给出正确答案 |
| 前缀的非语义效应 | Control 前缀（长度匹配、语义无关） |
| 题目难度混淆 | 混合效应回归中加入难度指标（步骤数 / 答案数值大小） |

## 阶段六：统计分析

### 主检验

| 对比 | 检验方法 |
|------|---------|
| 修正 vs 原始的最终准确率 | McNemar 检验（配对二元） |
| $\Delta$Acc 的置信区间 | 非参数 Bootstrap（10000 次） |
| 第2步修正的边际效应 | 混合效应 Logistic 回归 |
| 探针 AUC 差异 | DeLong 检验 / 配对 Bootstrap |
| 双重差分 $\Delta$ 显著性 | 置换检验（打乱修正/原始标签） |

### 回归模型细节

```
最终正确性 ~ 修正至第k步 + 步骤总数 + 题目难度 + 每步平均token数 + (1|题目ID)
```

重点关注"修正至第2步"的系数是否显著为正，且边际效应是否大于"修正至第1步"。
